📖 术语解释
📊 可视化
MDS Multidimensional Scaling 多维缩放。把 N×N 的 Jaccard 距离矩阵压缩成二维坐标，让你在平面上直观看到 run 之间的相似关系。两点越近 = 激活模式越相似。注意：MDS 是近似，远近关系保留大趋势，不完全精确。
Jaccard 距离 Jaccard Distance 衡量两个 run 激活了多少"不同"的神经元。公式：1 − |A∩B| / |A∪B|。0 = 完全相同，1 = 完全不同。这是 NAD 框架里最核心的距离度量。
KNN Purity K-Nearest Neighbor Purity KNN 纯度。对每个 run，找它在 Jaccard 空间中最近的 5 个邻居，统计其中 correct 的比例。均值 > 50% = 距离有正确性信号；≈ 50% = 激活模式与正确性无关。
Feature Space 特征空间散点图 以 dc_r（X 轴）和 reflection_count_r（Y 轴）为坐标画出所有 run。如果 correct（蓝）和 incorrect（红）能被一条线分开，说明这两个特征有判别力。
Activation MDS 激活空间 MDS 投影 以 Jaccard 距离矩阵做 MDS，显示神经元激活模式的分布。如果 correct 和 incorrect 在 MDS 图中聚成不同的团，说明激活模式本身有判别力。
🔢 特征
dc_r DeepConf rank DeepConf 置信度的组内排名，归一化到 [0, 1]。1 = 最自信，0 = 最不确定。底层是 tok_conf（token 级别置信度均值），值越低 = 模型越确定。
dc_z DeepConf z-score 同 dc_r，但用 z-score（均值 0，标准差 1）归一化。适合比较绝对量级，而不只是组内排名。
rc_r reflection_count_r 反思次数的组内排名，归一化到 [0, 1]。1 = 反思最多。"反思"指推理链中出现重启/修正信号（如 <think>、"wait"、"actually" 等模式）的次数，由轨迹分析提取。
rc reflection_count 原始反思次数（未归一化的整数）。
tok_conf Token Confidence 每个 token 的置信度指标（≈ 预测分布的熵的负数），值越小 = 模型越确定。注意方向：低 tok_conf → 高置信度。
tok_neg_entropy Token Negative Entropy 每个 token 预测分布的负熵（Σ p log p ≤ 0）。越接近 0 = 越确定（单峰分布）；绝对值越大 = 越不确定（均匀分布）。
🎯 Selector（选择算法）
Medoid 质心（几何中心） 选组内"最居中"的 run：到其他所有 run 的平均 Jaccard 距离最小的那个。代表共识最大的推理路径，理论上最"典型"。
KNN-Medoid K-Nearest Neighbor Medoid 选与最近 k 个邻居激活相似度最高的 run。比 Medoid 更关注局部聚集，对孤立异常 run 更鲁棒。
Min-Activation 最小激活 选激活神经元数量最少的 run（最"精简"的推理路径）。
Max-Activation 最大激活 选激活神经元数量最多的 run（最"丰富"的推理路径）。
DeepConf Deep Confidence 选 tok_conf 最低的 run（模型最确定的那条推理链）。核心假设：自信 = 正确。
Extreme8-Best Extreme 8-tuple Best ML 模型。从所有 run 中随机采样大量 8-tuple（8 个 run 的组合），用训练好的模型预测每个 run 是"最好"的概率（score_best），取均值后选最高分。训练信号来自 band_reward。
Extreme8-Mixed Extreme 8-tuple Mixed Best + Worst 双模型的混合版本，分数 = score_best − score_worst。既奖励"看起来好"，又惩罚"看起来差"，理论上更稳健。
band_reward Band Reward（带奖励） Extreme8 的训练信号。把 N 个 run 按某特征排序后分成 3 个 band（top/mid/bottom），统计每个 band 里 correct run 的数量，加权求和：α×top + β×mid + γ×bottom。前 band 中 correct 越多得分越高。这是 Extreme8 能否赢的核心假设。
📐 分数列说明
Score（归一化） Normalised Score 排名表中的 Score 列已归一化到 [0, 1]，1 = 该 selector 最偏好。注意：不同 selector 的 score 含义不同，只能在同一 selector 内比较，不能跨 selector 比较。
Rank 1 排名第 1 ⭐ 标记的那行就是该 selector 最终会选中的 run。如果 Rank 1 是红色 ✗，说明这个 selector 在这道题上选错了。
